# Company Templates

Terrain uses lightweight templates. The goal is to create enough shared structure that PM artifacts are easy to review without turning every document into a form.

---

## Document Templates

**PRD template**
- Problem
- Target user / segment
- Current state and evidence
- Goals and non-goals
- Proposed experience
- Requirements
- Success metrics
- Analytics / instrumentation
- Dependencies and risks
- Launch plan

**Ticket template**
- User story
- Context
- Acceptance criteria in Given/When/Then format
- Analytics events and properties
- Design link, if user-facing
- Dependencies
- Test notes

**Technical spec template**
- Context and product requirement link
- Proposed technical approach
- Alternatives considered
- Data model or API changes
- Rollout / migration plan
- Observability and failure modes
- Open questions

**Decision doc / RFC template**
- Decision to make
- Context
- Options considered
- Trade-offs
- Recommendation
- Decision owner
- Approvers / reviewers
- Date and revisit trigger

---

## Communication Templates

**Biweekly status update**
- Sprint goal and outcome: hit / partial / miss
- Current delivery health
- Risks or blockers
- What is changing next sprint
- Decisions needed from leadership

**Meeting notes**
- Objective
- Attendees
- Decisions made
- Open questions
- Action items with owners and dates
- Follow-up communication needed

**Launch comms**
- What is launching
- Who is affected
- Launch tier and rollout plan
- Support impact
- Known limitations
- Rollback trigger
- Owner and escalation path

**Stakeholder update**
- The point
- Why it matters now
- Current evidence
- Risks / trade-offs
- Ask or decision needed
- Timing

---

## Planning Templates

**Sprint plan**
- Sprint goal
- Capacity and availability
- Committed stories
- Stretch stories
- Dependencies
- Risks
- Validation / release notes

**Retro format**
- What changed since the last retro
- What worked
- What slowed us down
- Repeated patterns
- Action items with owners
- One process change to try next sprint

**Quarterly planning**
- Strategic priorities
- Candidate bets
- Impact / confidence / effort
- Dependencies and constraints
- Recommended sequence
- Explicit trade-offs
