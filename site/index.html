01 · What’s the kit

A portable, agentic system for the work PMs actually do.

The PM Agent Kit is a suite of product-management skills encoded as portable agents that run in Claude Code. It automates the work you do most: drafting specs, reviewing documents, planning sprints, synthesizing research, building business cases, and writing the leadership update at five o’clock.

This workbook is the structured practice companion. Each day introduces one skill, explains why it matters, and gives you two copy-ready prompts — a basic one and a richer scenario prompt anchored in Terrain, our fictional outdoor marketplace. github.com/nicoladevera/pm-agent-kit →

02 · How it works

Four moves. Repeated twenty times.

1

Set up the kit

Clone the repo, copy the Terrain context, run the Day 0 smoke-test. Roughly fifteen minutes once.

2

Meet Terrain

The fictional two-sided marketplace your prompts will reference for twenty days. Same product context, twenty lenses.

3

One skill per day

Pop the day page, copy a prompt, paste into Claude Code, read the output. Twenty minutes, then close the tab.

4

Reflect

Three questions to connect what you saw to your actual sprint. Notes live wherever you keep them.

04 · The form

“Interactive” means something specific.

Every prompt has a one-click copy button. Every day is linked. Each page carries machine-readable metadata for agents who want to load the curriculum as context. The site does not track progress, store state, gate days, or require login. Your notes live wherever you keep them.

The whole site is static HTML, no build step, no telemetry. You can read it on a plane.
